README

GraphMaker

GraphMaker is a Java application to create graphs consisting of nodes connected by directed and undirected edges, and perform various graph operations on them, such as breadth-first-search and minimum-spanning-tree.

Requirements

As of version 3.0, GraphMaker requires the Java Runtime Environment version 1.4 or better. It utilizes the preferences and XML parsing classes made available in the core library.

Starting GraphMaker

Starting the application amounts to invoking a small launcher. There are two possibilities at the moment, depending on your system.

Microsoft Windows

If you are using Microsoft Windows, you can double-click on the graphmaker.exe launcher in the bin directory, or invoke it from the command line.

Unix-like systems

On Unix-like systems such as Linux or Solaris, simply invoke the shell script graphmaker in the bin directory.

Startup options

The GraphMaker launcher is basically a renamed version of the NetBeans Platform launcher, thus it supports the same command-line options. For a list of these options and a brief explanation, invoke the launcher with the --help switch.

Documentation

The GraphMaker documentation consists of a User's Guide, the built-in help found in the Help menu in the main program window.

Removing GraphMaker

Removing GraphMaker from your system is generally pretty simple. Find the files, wherever it is that you originally put them, and delete them. If you used one of the packaged versions you should remove the package using the appropriate system command. In addition to the program files, there are user settings stored in a directory named .graphmaker in your home directory.

Upgrading the JDK

If at some point you replace your current JDK installation with a different version, you will need to update the GraphMaker configuration file. This is done by opening the etc/graphmaker.conf file in a text editor and changing the netbeans_jdkhome value to point to the new JDK installation directory. An alternative to editing the configuration file is to pass the --jdkhome option when starting GraphMaker. This option takes a single argument, which is the path to the JDK installation.
