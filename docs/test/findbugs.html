FindBugs

FindBugs is a static analysis tool that operates on the bytecode of your Java appliation to identify bugs in the code itself. Some of the "bugs" it finds are not so important, but many of them are worth evaluation. On average, the false positive rate is low enough to make using the tool worth while.

Using it to Find Bugs

Using FindBugs on GraphMaker is pretty straight forward. Launch the FindBugs program and the main window should appear. In the project settings window, specify the root of the GraphMaker source tree (e.g. core/src), the build directory (e.g. core/build), and the .jar files that are included in the classpath for that module (e.g. platform5/core/org-openide.jar, et cetra). See the module build.xml file for the classpath. Once the project settings are defined, click the "Find Bugs" button to have the tool perform its analysis.

An even simpler method of using FindBugs is to get the NetBeans plugin, available here. Once the NetBeans plugin is installed, just select the GraphMaker project and invoke the FindBugs action from the main toolbar in the NetBeans IDE.

There are some bugs that should not be fixed. These are either not really bugs (i.e. a string comparison using the equals operator is not always wrong), or they are bizarre in some way or another and I cannot understand what the tool is trying to tell me (e.g. an obscure runtime exception that is not explicitly caught, such as NoSuchFieldException). In general though, there should be very few problems overall.
